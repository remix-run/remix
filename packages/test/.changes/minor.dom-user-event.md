Add a `@remix-run/test/dom` subpath export with a synthetic `userEvent` helper for browser-mode tests. Exports `userEvent.type(element, text)` (per-character keydown / beforeinput / input / keyup, mutating `<input>` / `<textarea>` value at the cursor), `userEvent.press(element, key)` (single keydown / keyup pair), and `userEvent.click(element, options?)` (click MouseEvent honoring `MouseEventInit` — auto-dispatches `auxclick` for non-primary buttons). The `press` key argument is typed as `KeyboardKey`, which autocompletes the common named keys (`Enter`, `Escape`, `ArrowDown`, etc.) while still accepting arbitrary single-character keys.

The DOM-only helpers live in their own subpath so importing from `@remix-run/test` keeps the main entry's types node-compatible — server-only test files don't get DOM lib types pulled in via `TestContext`.
