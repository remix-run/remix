Allow `createAssetServer()` base paths to include optional URL-only params, such as `basePath: '/(:version/)assets'`, so a single asset server can serve the same files from both versioned and unversioned URL namespaces while still generating unversioned asset URLs from file paths. `assetServer.getHref()` and `assetServer.getPreloads()` now accept typed base path params, such as `{ version }`, to generate hrefs under a specific base path variant. When scripts or styles are requested through a resolved optional base path, compiled import, source map, and CSS file URLs preserve that matched base pathname.
