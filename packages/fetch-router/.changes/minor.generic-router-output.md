Add an augmentable `RouterTypes.output` contract to `createRouter`. Routers can now return application values such as `RemixNode`, with the output type propagated through middleware, controllers, mounted routes, and nested `context.router.fetch` calls. Middleware that explicitly requires `Response` output is rejected by custom-output routers. Existing `string | URL | Request` inputs and cancellation through `router.fetch(url, { signal })` remain unchanged.
