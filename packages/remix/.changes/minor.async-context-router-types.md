BREAKING CHANGE: `remix/async-context-middleware` no longer exposes `AsyncContextTypes`. `getContext()` now derives its type from `remix/fetch-router`'s `RouterTypes.context`, with route params broadened to `AnyParams`, so apps only need the router context augmentation. `MiddlewareContext` and `ContextWithMiddleware` also accept middleware factory function types, so app context tuples can use entries like `typeof session` instead of `ReturnType<typeof session>`.
