Added database wipe, migration, status, seed, and reset APIs through `remix/data-table` and `remix/data-table/cli`. `Database.migrate()` supports forward and backward directions, targets, steps, dry runs, and custom journal tables. `runRemixDb()` accepts resolved runtime values, and migration status reports applied migrations whose files are missing without creating an absent journal table. `Database.close()` releases adapter-owned connection pools, and `loadSeed()` loads a SQL seed file as a `Seed` function (see #11608, #11639).

BREAKING CHANGE: Removed the public migration runner factory and types. Use `Database.migrate()` and `Database.migrationStatus()` instead.
