Added concrete `createPostgresDatabase()`, `createMysqlDatabase()`, and `createSqliteDatabase()` factories alongside database wipe, migration, status, seed, and reset APIs through `remix/data-table` and `remix/data-table/cli`. `Database.migrate()` supports forward and backward directions, targets, steps, dry runs, and custom journal tables. `runRemixDb()` accepts resolved runtime values, and migration status reports applied migrations whose files are missing without creating an absent journal table. Concrete databases release owned resources with `close()`, and `loadSeed()` loads a SQL seed file as a `Seed` function (see #11608, #11639).

BREAKING CHANGE: Removed public database adapter composition and the migration runner factory/types. Create a complete dialect-specific database with `createPostgresDatabase()`, `createMysqlDatabase()`, or `createSqliteDatabase()`, then use `Database.migrate()` and `Database.migrationStatus()` for migrations.
