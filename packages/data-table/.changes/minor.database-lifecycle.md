Added `Database.wipe()`, `Database.migrate()`, `Database.migrationStatus()`, and `Database.reset()` along with the typed `runRemixDb()` API from `remix/data-table/cli`. Migration status reports applied journal entries missing from the current migration set, and migration runs reject missing applied migrations before executing SQL. Migration adapters can use the scoped `withMigrationLock()` contract to keep execution on the connection that owns the lock. Apps can export `db`, `getMigrations`, and an optional `seed` function from `app/db.ts` for use by `remix db` (see #11608).
