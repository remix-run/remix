Added `Database.wipe()`, `Database.migrate()`, `Database.migrationStatus()`, and `Database.reset()` along with the typed `runDataTableCommand()` API from `remix/data-table/cli`. Migration adapters can use the scoped `withMigrationLock()` contract to keep execution on the connection that owns the lock. Apps can export `db`, `getMigrations`, and an optional `seed` function from `app/db.ts` for use by `remix db` (see #11608).
